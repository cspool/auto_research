# H2.2 Protocol: Triton Compiler Metadata for Variant Scheduling

Date: 2026-05-26

Status: protocol locked in file. Git pre-registration is unavailable because
`/data3/auto_research` is not a git repository.

## Hypothesis

H2: Micro-operator tiling plus multi-version kernels improves scheduling.

This H2.2 follow-up narrows the question to compiler/runtime interface:

> Triton already emits enough compile-time metadata to identify variant resource
> knobs such as tile size, warp count, thread block size, shared memory, and PTX
> register declarations. This metadata is not a full profiler substitute, but it
> is a practical basis for runtime scheduling metadata.

## Input

Use the Triton cache generated by H2.1 for `fused_gelu_residual_kernel`.
For each cache entry, inspect:

- `fused_gelu_residual_kernel.json` for target architecture, `num_warps`,
  `num_ctas`, `shared`, and Triton version;
- `fused_gelu_residual_kernel.ttgir` for tile tensor size and `num-warps`;
- `fused_gelu_residual_kernel.ptx` for `.reqntid` and `.reg` declarations;
- optional file sizes for PTX/cubin as a rough codegen footprint signal.

## Metrics

| Metric | Meaning |
|---|---|
| `block_size` | Parsed Triton tile size from TTGIR tensor shapes. |
| `num_warps` | Triton CTA warp count from JSON/TTGIR. |
| `reqntid` | CUDA thread block size from PTX. |
| `shared_bytes` | Static shared memory bytes from Triton metadata. |
| `reg_*_decl` | PTX register declaration counts by register class. |
| `ptx_bytes` / `cubin_bytes` | Codegen footprint proxy. |

## Prediction

If the compiler metadata path is useful:

- every measured variant can be recovered from cache artifacts;
- metadata will expose at least tile size and warp/thread count without Nsight;
- latency differences can be joined with metadata to produce a scheduler table.

If it is not useful:

- variant tile sizes or resource knobs cannot be reliably recovered;
- important resource pressure signals are absent from compiler artifacts.

## Sanity Checks

- Do not require GPU execution; this is a cache-artifact analysis.
- De-duplicate cache entries by `(n_elements, block_size, num_warps)`.
- Treat PTX register declarations as a coarse static signal only, not as
  achieved occupancy.
