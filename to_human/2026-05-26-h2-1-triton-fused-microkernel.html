H2.1: 融合微算子的并发 variant 选择

Triton fused epilogue · RTX 4090 · 2026-05-26

阶段性结论： H2 得到加强。把 vector add 换成 GELU(X * scale + Bias) + Residual 后，Triton fusion 明显快于 PyTorch eager unfused chain；同时，最佳 standalone variant、最佳 concurrent latency variant、最大 stream speedup variant 仍会分离。

关键结果

Shape PyTorch unfused Best fused Best isolated Best concurrent Best speedup
balanced 2048 0.2721 ms 0.1422 ms 2048 / 8 2048 / 4, 0.2271 ms 4096 / 4, 1.2000x
memory-heavy 1024 1.4640 ms 0.5614 ms 256 / 8 512 / 4, 0.5726 ms 2048 / 4, 1.1427x

解释

Fusion 解决的是 launch 和 global-memory 往返；concurrent scheduling 解决的是 在同一张卡上如何把不同资源压力的微算子放在一起。这两个问题相互补充。 balanced shape 中，如果只按 standalone fused latency 选 2048/8， 并发延迟是 0.2766 ms；按 concurrent latency 选 2048/4， 延迟降到 0.2271 ms。

memory-heavy shape 给了另一个提醒：最大 stream speedup 不等于最低绝对延迟。 2048/4 的 speedup 最高，但 concurrent latency 为 0.5920 ms， 比最佳 concurrent variant 的 0.5726 ms 更慢。
