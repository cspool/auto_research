外循环 1：从 stream overlap 到 compiler/runtime variant scheduling

5 个实验后的研究方向收敛 · 2026-05-26

当前主张： 单请求、单加速器的多算子并发，不是单纯的 stream API 问题，而是 compiler/runtime co-scheduling 问题。关键单位是带多版本实现和 metadata 的 micro-operator。

证据链

实验 结论
H1 naive CUDA streams 只有 -0.3% 到 +4.3% speedup，overlap 接近零。
H1.1 PyTorch row chunking 太粗，会改变 GEMM kernel 选择并引入 launch overhead。
H2 Triton vector-add 的 best isolated variant 不等于 best concurrent variant。
H2.1 融合 epilogue 中，fusion 很有用，但 variant scheduling 仍然必要。
H2.2 Triton cache 可恢复 tile/warps/thread/register/code-size metadata。

方法框架

推荐框架是：模型语义分解 → 多版本微算子生成 → compiler metadata 保留 → empirical/profiler-guided co-scheduling → 以 absolute concurrent latency 为主目标。

下一步
