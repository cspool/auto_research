Outer Loop 3: sparse Video/VLM KV retrieval

H4.1-H4.3 · single request · RTX 4090 · 2026-05-26

综合结论： Video/VLM 稀疏 retrieval 的有效执行单元不是 per-frame/per-cluster framework loop，而是 fused sparse attention fragment： selected-token movement、score、softmax/reduction 和 value aggregation 应该被编译器可见地融合。

证据链

阶段 结果 判断
H4.1 segment loop fused Triton 快 52x-64x semantic segment 是 metadata，不是 launch unit
H4.1 flat gather-score fused Triton 快 1.15x-1.35x K materialization 可避免
H4.2 order sweep fused Triton 快 1.37x-1.42x order 要和 tile variant 一起选
H4.3 score-softmax-value fused Triton 快 5.78x-6.79x movement + reduction + value aggregation 融合是关键
