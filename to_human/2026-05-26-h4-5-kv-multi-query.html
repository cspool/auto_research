H4.5: multi-query sparse attention

1024 selected tokens · Q=4 shared K/V · RTX 4090 · 2026-05-26

阶段性结论： grouped-query Triton 支持 H4。把 4 个 query 放进同一对 online-softmax kernels 后，比 flat PyTorch 快 2.86x 到 3.64x， 比逐 query 重复执行 H4.4 式 kernel 快 3.41x 到 3.66x。

解释

H4.5 的关键不是“又写了一个更快的 kernel”，而是把 query/head 复用变成 compiler-visible 的调度单位。逐 query 重复执行会重新付 launch、K/V 读、 score 和 partial reduction 成本，最后大致退回 flat PyTorch 的延迟区间； grouped-query kernel 则把这些工作摊到 4 个 query 上。

结果还给了一个清晰的资源边界：N256 D64 V64 Q4 W4 在所有 full runs 中 shared memory 超限，要求 133120 bytes，而硬件上限是 101376 bytes。 这说明 H4 的 runtime table 必须同时包含 measured latency 和 static resource feasibility。
