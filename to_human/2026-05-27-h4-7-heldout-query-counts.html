H4.7: held-out query counts

Q=2 and Q=8 · grouped sparse attention selector · 2026-05-27

阶段性结论： H4.6 的 runtime table 不是只会复述 H4.5。 在 Q=2/Q=8 的 8 个 held-out cases 上，静态过滤继续拒绝 8/8 个 OOR rows； order-aware selector 在 7/8 cases 内落到 measured best 的 15% 以内，平均 regret 低于 global mean selector。

摘要

Selector Near-best cases Mean regret Max regret
global mean latency 7/8 9.92% 27.44%
order-aware rule table 7/8 7.79% 18.72%

关键失手

Case Selected Measured best Regret
Q=8 clustered segment N128 D64 V128 Q4 W4 N128 D64 V64 Q4 W8 18.72%

解释

H4.7 支持 runtime-table 方向，但也指出了下一个缺失特征：query-count pressure。 Q=8 时同样的 clustered order 不再偏好 H4.6 的 V128/W4，而偏好 V64/W8。 这意味着下一版 selector 应把 query_count、num_q_blocks 和 query_count / BLOCK_Q 纳入表项。
