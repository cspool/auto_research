H2: 最快 kernel variant 是否等于最佳并发 variant？

Triton variant scheduling · RTX 4090 · 2026-05-26

阶段性结论： H2 获得初步支持。两个 shape 中，最快 isolated Triton add variant 都不是最低 concurrent latency 的 variant。runtime 不能只看 standalone latency，必须知道调度上下文。

结果

Run Best isolated Add ms Best concurrent Concurrent ms Overlap
balanced 2048 block=4096, warps=4 0.1019 block=2048, warps=8 0.1864 0.1652
memory-heavy 1024 block=256, warps=8 0.4227 block=128, warps=4 0.4349 0.0197

为什么重要

H1 和 H1.1 说明 naive stream 和粗粒度 chunking 不够。H2 的结果进一步说明： 需要多版本 micro-kernel，但 runtime 选择目标不能只是“孤立最快”。同一个 Triton micro-op 在 isolated latency、concurrent latency、stream speedup、overlap 上会有不同最优点。

下一步

H2.1 应从 vector add 走向 model-shaped fused micro-kernel，例如 add + activation + scale、dequantize + add 或小 expert FFN tile， 并加入 Nsight/带宽指标，把 variant 差异连接到具体资源压力。
