H2.2: Triton 编译产物能给 scheduler 什么 metadata？

Compiler/runtime interface · RTX 4090 Triton cache · 2026-05-26

阶段性结论： 没有 Nsight 工具时，Triton cache 仍能恢复每个 fused micro-kernel variant 的 tile size、warps、thread block、shared memory、 PTX register declaration 和 code size。这不足以解释全部性能，但足够构成 runtime variant table 的静态 metadata 部分。

解释

静态 metadata 不能单独预测 latency，但它给 runtime 一个实际可实现的接口： 每个 generated variant 都可以带上 tile/warps/register/code-size 等特征，再 结合经验测量或 profiling，按 absolute concurrent latency 选择 co-scheduling variant。

这把 H2/H2.1 的 latency 现象接到了编译框架实现：不是只生成一个 fused kernel， 而是生成多版本 kernel，并保留能被调度器消费的 variant table。
